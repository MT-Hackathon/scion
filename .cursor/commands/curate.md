# curate

The environment is our shared memory. Everything in it costs tokens on every future session — bad information actively misleads, stale content silently degrades judgment. The bar for adding is high. The bar for pruning is not as high but not low either.

Start by reading what's already there. The default question is not "what should we add?" but "what should we refine, prune, or correct — and does anything new genuinely earn its place?" Integration into existing artifacts is always preferred over creating new ones.

Use the Curator to evaluate against the quality rubric: token efficiency, placement correctness, overlap detection, mechanism-teaching over order-stacking. Use the Author to write. Bring dispositions to me for approval — I'm a decision partner here, not a reviewer at the end. We evaluate together, specialists write.
