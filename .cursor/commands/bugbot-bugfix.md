# bugbot-bugfix

Have a researcher pull the comments from the open MR in origin — the JSON returns eat context for no reason if read directly. Use explorers to confirm the presence or sprawl of the errors.

Design with as many Architects as the problem needs. Then create a plan that delegates: code work to Executor, documentation work to Author, and code review iteration to QA. Maximize efficiency while keeping the orchestrator's context window clear enough to evaluate every agent return critically rather than accepting it at face value.

Include visual QA steps when the fix directly changes visual components. Once the team has fixed it, have Author write a comment to the MR — but do not close it.

The goal is not just passing tests. The question is whether this is genuinely great code. Would you get on the space shuttle this code is running. The team collaboration is how we get there.
