# Checklist: Pre-Change

Verify before making any code changes.

---

## Search Verification

- [ ] Have I searched for existing implementations of this logic?
- [ ] Have I read at least 10 lines above and below the target code?

## Plan Quality

- [ ] Is my plan prescriptive and free of vague language?
- [ ] Have I asked the user to clarify any ambiguities?

## Code Quality

- [ ] Is the code "NASA Grade" (no hacks, no magic numbers)?
- [ ] Am I fixing the underlying code instead of workarounds in tests?
- [ ] Is the plan stable (not renamed since creation)?

## Prohibited Actions

- Testing without reading code first
- Creating new modules without thorough searching
- Unrequested TODOs or shortcutting tests
