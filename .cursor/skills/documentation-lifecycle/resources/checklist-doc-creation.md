# Checklist: Documentation Lifecycle

Verify these organization mandates before creating new documentation.

---

## Creation Checklist

- [ ] Is this file outside the project root?
- [ ] Is it in the correct standardized location (`.cursor/plans/`, `docs/...`)?
- [ ] Does the filename use kebab-case?
- [ ] If time-sensitive, does it have the `YYYY-MM-DD-` prefix?

## Maintenance Checklist

- [ ] Has this document been referenced in related issues/PRs?
- [ ] Is this document still accurate and up-to-date?
- [ ] (Quarterly) Should this document be archived or deleted?
