# Value-Effort Matrix Template

Use this template to prioritize capabilities and features.

## Prioritization Matrix

| Capability | Value (User/Business) | Effort (Technical/Design) | Priority | "Buy-In" Rationale |
| :--- | :--- | :--- | :--- | :--- |
| [Feature A] | High | Low | **P1 (Quick Win)** | [Why it matters] |
| [Feature B] | High | High | **P2 (Strategic)** | [Why the effort is worth it] |
| [Feature C] | Low | Low | **P3 (Fill-in)** | [When to do it] |
| [Feature D] | Low | High | **Avoid (Sinkhole)** | [Why we shouldn't build it] |

## Minimum Viable Experience (MVE)
Define the smallest set of features that provide enough value to justify the user's effort.

- **Must-Have**: [Non-negotiable for the core journey]
- **Should-Have**: [Important but the journey survives without it]
- **Could-Have**: [Nice to have, adds polish]

## Risk-Adjusted Sequencing
- **Technical Risk**: [Dependencies on new/unstable tech]
- **Adoption Risk**: [Will users actually change their behavior?]
- **Sequence**: [1. Solve the hardest problem, 2. Build the foundation, 3. Iterate on polish]
