# Checklist: Git Workflows

Validation steps for git operations (GitHub and GitLab).

---

## Commit Checklist

- [ ] Conventional commit format used
- [ ] Clear and descriptive message
- [ ] Related issue referenced
- [ ] Code reviewed before commit

## Branch Checklist

- [ ] Branch named descriptively
- [ ] Up to date with main/develop
- [ ] Tests passing before merge

---

## GitLab Pipeline Checklist

- [ ] Pipeline configuration valid
- [ ] Jobs properly sequenced
- [ ] Artifacts configured
- [ ] Deploy gates in place

## Merge Request Checklist

- [ ] Description clear and complete
- [ ] Related issue linked
- [ ] Tests passing
- [ ] Reviewers assigned
- [ ] No merge conflicts

## Issue Management Checklist

- [ ] Issue properly labeled
- [ ] Milestone assigned
- [ ] Epic linked (if applicable)
- [ ] Priority set
